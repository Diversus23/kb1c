При записи пользователя (или внешнего пользователя) с установленным флажком Доступ к информационной базе разрешен, вместе с ним автоматически создается соответствующий ему новый (или модифицируется существующий) пользователь ИБ. И наоборот, когда пользователь (или внешний пользователь) записывается со снятым флажком Доступ к информационной базе разрешен, то соответствующий ему пользователь ИБ автоматически удаляется из списка пользователей ИБ. Однако есть ряд ситуаций, когда список пользователей ИБ становится рассинхронизированным со списком пользователей. Тогда с помощью данного списка могут быть решены следующие задачи:

  • Найти "некорректных" пользователей ИБ и синхронизировать списки пользователей ИБ и пользователей между собой:
    • если пользователь ИБ был добавлен с помощью конфигуратора (или каким-либо нештатным программным способом), предлагается удалить такого пользователя ИБ;
    • если пользователь ИБ был удален с помощью конфигуратора (или каким-либо нештатным программным способом), но при этом осталась связанная с ним запись в списке пользователей, предлагается открыть и перезаписать сведения о пользователе, для того чтобы очистить связь с удаленным пользователем ИБ;
    • если полное имя пользователя ИБ было изменено с помощью конфигуратора (или каким-либо нештатным программным способом), предлагается открыть и перезаписать сведения о пользователе, для того чтобы восстановить полное имя пользователя ИБ.
  • Посмотреть список пользователей и внешних пользователей, для которых не существует соответствующих пользователей ИБ (в случае если у пользователя не установлен флажок Доступ к информационной базе разрешен).
  • Посмотреть список всех пользователей ИБ (этот список также доступен в конфигураторе).

 

Для того чтобы открыть этот список, можно воспользоваться командой Пользователи информационной базы в меню Все действия списка пользователей (или внешних пользователей).
