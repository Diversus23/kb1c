Для доступа в Интернет с клиентских компьютеров параметры прокси‑сервера задаются на каждом клиентском рабочем месте (по команде Персональные настройки в разделе Настройка и администрирование).

При использовании клиент‑серверной версии 1С:Предприятия параметры прокси‑сервера также указываются и для сервера 1С:Предприятия (форма настроек параметров системы в разделе Настройка и администрирование). В общем случае, параметры прокси-сервера для клиентских рабочих мест и сервера 1С:Предприятия могут различаться.

Настройка параметров прокси-сервера для механизмов, расчитанных на работу с веб-сервисами
При обращении к веб-сервисам параметры прокси-сервера зачитываются автоматически из специального конфигурационного файла "inetcfg.xml", расположенного в каталоге конфигурационных файлов платформы 1С:Предприятие. Каталог конфигурационных файлов платформы 1С:Предприятие, как правило, располагается по адресу "<Каталог установки платформы 1С:Предприятие>\conf\", например "C:\Program Files\1cv82\8.2.16.362\bin\conf\". Если в файле "<Каталог установки платформы 1С:Предприятие\conf\conf.cfg" задан параметр "ConfLocation=<Каталог конфигурационных файлов>", то конфигурационные файлы платформы 1С:Предприятие располагаются в каталоге <Каталог конфигурационных файлов> (как правило, это "C:\Program Files\1cv82\conf\" или "C:\Program Files\1cv8\conf\").

При настройке параметров прокси-сервера для механизмов, расчитанных на работу с веб-сервисами:
- в файловом режиме работы информационной базы требуется на каждом компьютере пользователя записать настройки прокси-сервера файл inetcfg.xml (может быть выполнено при сохранении пользователем персональных настроек прокси-сервера).
- в клиент-серверном режиме работы информационной базы необходимо, чтобы настройки прокси-сервера были записаны в файл inetcfg.xml на каждом компьютере рабочего сервера 1С:Предприятия, входящего в состав кластера. Если в кластере только один рабочий сервер 1С:Предприятия, то настройки прокси-сервера могут быть записаны в файл при сохранении администратором настроек прокси-сервера для сервера 1С:Предприятия, в противном случае требуется сохранить настройки прокси-сервера в файле (Сохранить как...) и скопировать сохраненный файл в соответствующий каталог конфигурационных файлов платформы 1С:Предприятия на каждом копьютере рабочего сервера кластера.

Особенности использования конфигурационного файла "inetcfg.xml":
1) настройки из файла "inetcfg.xml" действуют для всех информационных баз, работа с которыми осуществляется на версиях платформы 1С:Предприятия, использующих один файл inetcfg.xml;
2) имя пользователя и пароль от прокси-сервера хранятся в открытом виде;
3) настройки, заданные в файле, будут по умолчанию использованы другими механизмами платформы 1С:Предприятия, если для этих механизмов не указаны явные настройки прокси-сервера;
4) при необходимости использования платформой 1С:Предприятие системных настроек прокси-сервера файл "inetcfg.xml" необходимо удалить из соответствующего каталога.
