Какие виды панелей навигации доступны списка взаимодействий?
  • По предметам - панель навигации будет заполнена предметами взаимодействий. При позиционировании на предмете, в списке будут отображаться взаимодействия по данному предмету.
  • По контактам - панель навигации будет заполнена контактами, с которыми были взаимодействия. При позиционировании на контакте, в списке будут отображаться взаимодействия с данным контактом.
  • По папкам электронных писем - панель навигации будет доступными пользователю учетными записями и их папками. При позиционировании на папке в списке будут отображаться письма данной папки.
  • По закладкам - панель навигации будет заполнена закладками пользователя. При позиционировании на закладке в списке будет отображаться те взаимодействия, которые соответствуют отбору, настроенному в закладке.
  • По свойствам - для каждого дополнительного реквизита или сведения документов взаимодействий формируется отдельный вид панели навигации. При выборе такого вида навигации панель заполняется всеми возможными значениями данного дополнительного реквизита или сведения. При позиционировании на одном из значений, в списке взаимодействий отображаются только те документы, у которых соответствующее сведение имеет такое же значение.
  • По категориям - панель данного вида заполняется видами свойств документов взаимодействий, значениями которых могут быть либо Истина либо Ложь. При позиционировании на виде свойства в панели навигации в списке взаимодействий отображаются только те документы, у которых значение данного свойства Истина.

Как переключить вид панели навигации?


Воспользовавшись выпадающим меню, которое располагается над панелью навигации.

Какие отборы можно установить для списка, помимо отбора наложенного в панели навигации?
  • Тип взаимодействия - возможно установить отбор по определенному типу взаимодействий, например по входящим письмам или телефонным звонкам.
  • Ответственный - возможно установить отбор по ответственному за отработку взаимодействий.
  • Статус - возможно отобрать взаимодействия с определенным статусом рассмотрения, например только не рассмотренные, или только отложенные.
  • Период - возможно установить интервал дат документов взаимодействий.

Можно ли найти взаимодействие при помощи полнотекстового поиска?


Да, можно, для этого надо ввести строку поиска в поле, расположенное в правой верхней части формы. Будет выполненный поиск по текстам, содержащимся в документах взаимодействий.

Данный поиск позволяет найти взаимодействия, для которых либо в самой форме, либо в текстах присоединенных файлов найдена строка, удовлетворяющая условиям.

После выполнения поиска в списке взаимодействий остаются только те, которые удовлетворяют условию поиска. При позиционировании на взаимодействии внизу на форме отображается информация о тексте, который совпадает с условием поиска.

В случае если ни одно взаимодействие найдено не будет, поле ввода строки поиска будет подсвечено красным.

Для того чтобы отменить отображение списка по результатам полнотекстового поиска необходимо очистить строку поиска.

Важно. Следует учитывать, что некоторые символы, введенные в строку поиска, будут трактоваться как служебные символы языка запросов полнотекстового поиска в данных. Информацию о списке таких служебных символов можно получить в справке (раздел 1С:Предприятие - Работа пользователя - Язык запросов полнотекстового поиска в данных).

Может ли пользователь инициировать процесс отправки и получения писем?


В файловом варианте хранения информационной базы это возможно. Для этого можно воспользоваться командой Получить и отправить, расположенной в командной панели списка взаимодействий.
В клиент-серверном варианте хранения информационной базы отправка и получение писем выполняются с определенным администратором системы интервалом.

Можно ли ответить или переслать письмо из списка взаимодействий?


Да, если вид панели навигации По папкам электронных писем. Если выполнено позиционировании на папке, в командной панели списка взаимодействий появляются соответствующие назначению папки команды. Например, для папки Входящие, будут доступны команды Ответить, Ответить всем, Переслать. При выполнении одной из этих команд будет выполнено соответствующее действие с письмом, на котором выполнено позиционирование в списке взаимодействий.

Для просмотра отложенных взаимодействий используется статус Отложенные. При установке этого статуса в списке отображаются все неотработанные взаимодействия, у которых заполнена информация в поле На контроле или Рассмотреть после.

Как установить признак рассмотрено для нескольких документов в списке?


Выделите документы, у которых необходимо установить признак Рассмотрено. Воспользуйтесь командой Рассмотрено (Ctrl+R) командной панели списка.

Для групповой установки признака Рассмотрено можно воспользоваться соответствующей командой контекстного меню панели навигации. В этом случае признак Рассмотрено будет установлен для всех документов, удовлетворяющих отбору списка. Например, при выполнении данной команды на папке Входящие, будет установлен признак Рассмотрено для всех писем данной папки.

Как снять признак рассмотрено для нескольких документов в списке?


Выделите документы, у которых необходимо снять признак Рассмотрено. Воспользуйтесь командой Снять признак рассмотрено (Alt+Shift+R), расположенной в меню Все действия списка взаимодействий.

Как отложить рассмотрение нескольких документов в списке?


Выделите документы, рассмотрение которых необходимо отложить. Воспользуйтесь командой Отложить рассмотрение командной панели.

Отложить рассмотрение несколько взаимодействий можно также воспользовавшись соответствующей командной контекстного меню панели навигации. В этом случае будет отложено рассмотрение взаимодействий, удовлетворяющих отбору списка. Например, при выполнении данной команды на папке Входящие, будет отложено рассмотрение всех писем данной папки.

Как установить предмет взаимодействий для нескольких документов в списке?


Выделите документы, у которых необходимо установить новый предмет. Воспользуйтесь командой Установить предмет взаимодействия (Ctrl+G) командной панели списка.

Для групповой установки предмета взаимодействий можно воспользоваться соответствующей командой контекстного меню панели навигации. В этом случае выбранный предмет будет установлен для всех документов, удовлетворяющих отбору списка. Например, при выполнении данной команды на папке Входящие, будет установлен указанный предмет для всех писем данной папки.

Как назначить ответственного для нескольких документов в списке?


Выделите документы, у которых необходимо установить ответственного. Воспользуйтесь командой Установить ответственного (Ctrl+M) командной панели списка.

Для групповой установки ответственного можно воспользоваться соответствующей командой контекстного меню панели навигации. В этом случае ответственный будет установлен для всех документов, удовлетворяющих отбору списка. Например, при выполнении данной команды на папке Входящие, будет установлен выбранный ответственный для всех писем данной папки.

Как переместить письмо из одной папки в другую?


Выделите одно или несколько писем. Воспользуйтесь командой Переместить в папку контекстного меню списка. Выберите папку, в которую необходимо перенести письма.

Если установлен вид навигации По папкам электронных писем то для перемещения писем в другую папку можно использовать перетаскивание. Выделите одно или несколько писем и перетащите их на папку в панели навигации.

Как создать закладку взаимодействий из данной формы?


Достаточно выделить в списке одно или несколько взаимодействий и воспользоваться командой Добавить в закладки контекстного меню списка. Будет создана закладка, включаюящая в себя отбор по выбранным взаимодействиям

Для создания закладки можно воспользоваться соответствующей командой контекстного меню панели навигации. В этом случае закладка будет создана для всех документов, удовлетворяющих отбору списка. Например, при выполнении данной команды на папке Входящие, будет создана закладка для всех писем данной папки.
